# 03. Bug Fixes & Improvements Log

## 1. 메인 화면 흰색(백화) 렌더링 오류 해결
- **증상**: 온보딩 완료 후 메인 홈 화면 진입 시 하단 네비게이션(BottomNav)만 보이고, 메인 콘텐츠 영역이 전체가 하얗게 나타나는 오류 발생.
- **원인**: 
  - `src/app/page.tsx` 파일 내에서 다국어 작업 및 UI 구조 변경 시, `lucide-react` 모듈에서 화면 표시에 필요한 아이콘(`Plus`, `Book`, `ArrowRight`)의 import를 누락함.
  - 이로 인해 렌더링 과정에서 `ReferenceError`가 발생하여 `page.tsx` 전체의 렌더링이 중단됨(React의 Error Boundary나 Fallback이 없어서 상위 레이아웃을 제외하고 백화 현상).
  - (추가적으로 `page.tsx` 하단에 레이아웃에 이미 존재하는 `<BottomNav />`, `<AdBlockModal />` 컴포넌트가 중복 삽입되어 `ReferenceError: BottomNav is not defined`가 발생한 것도 연관된 렌더링 오류 원인이었음)
- **해결 방안**: 
  1. 누락되었던 아이콘 컴포넌트를 import 목록에 추가.
  2. `page.tsx`를 기존 원본 UI로 복구하여 중복 삽입된 태그 및 레이아웃 충돌 문제를 해소함.

## 2. 캐릭터 생성 직후 선톡(초기 메시지) 미발송 버그 해결
- **증상**: 캐릭터 생성 직후 홈/채팅 목록에서는 "새로운 메시지가 있어요"라고 뜨지만, 막상 채팅방에 들어가보면 캐릭터의 선톡 메시지가 존재하지 않는 현상.
- **원인**: 
  - 최근 서버 사이드 채팅 API(`api/chat/route.ts`) 업데이트 과정 중, 메시지 저장 로직에 `requestId` 필드가 새로 추가됨.
  - 최초 선톡 생성 시에는 `requestId`가 `undefined` 상태로 전달되는데, Firestore의 `setDoc()` 함수는 `undefined` 값을 지원하지 않으므로 백그라운드 DB 저장 작업 중 `Unhandled Runtime Error`를 뱉으며 조용히 실패함.
  - 결과적으로 OpenRouter API를 통해 응답은 정상적으로 받아왔으나 DB 저장이 누락되었음. 반면 UI 목록은 데이터가 없을 때 기본 텍스트(`chat.newMessage`)를 띄우고 있어 마치 메시지가 온 것처럼 보였던 것임.
- **해결 방안**: 
  - `src/app/api/chat/route.ts`의 DB 저장 객체(`newMsg`) 생성 로직을 수정.
  - `requestId`가 존재할 때만 객체 프로퍼티에 포함되도록 예외 처리(`...(requestId ? { requestId } : {})`)를 추가하여 Firestore가 에러 없이 정상 저장하도록 조치 완료.

## 3. "내 캐릭터 설정" 진행 중 뒤로가기 동작 개선
- **개선 사항**: `onboarding` 내 캐릭터 설정 단계(이름, 성별 등 입력) 진행 중, 이전 화면으로 돌아가기 위해 `<ChevronLeft>`(뒤로가기) 버튼 클릭 시의 동작 수정.
- **상세 내역**: 
  - 기존에는 무조건 전 단계(예: `userStep 2` -> `userStep 1`)로 넘어가거나 설정 알림창으로 돌아갔음.
  - 이제 `userStep 1`에서 뒤로가기 버튼 클릭 시 사용자 의도를 재차 확인하는 전용 경고 모달 표시.
  - **모달 타이틀**: "뒤로가면 작성한 내용이 사라져요"
  - **버튼 구성**: (비율 3:7)
    - [뒤로가기] - 빨간색 계열의 경고 색상 적용 (`rgba(255, 59, 48, 0.1)` 배경, `#FF3B30` 텍스트). 클릭 시 기존 입력 정보를 모두 초기화하고 설정 스킵 로직 실행.
    - [계속하기] - 기존 포인트 컬러 적용. 클릭 시 모달창을 닫고 다시 작성 단계로 복귀.
